# Portfolio Gallery Design

日期：2026-04-18

## 目标

把 `Cosoul.AI` 主项目区和四个子项目截图区从静态单图/多图直排，改成统一的“单图显示 + 手动切换”展示模式。

## 问题

当前页面存在两个明显问题：

1. `Cosoul.AI` 主项目区右侧是 `2 张小图 + 1 张大图`，视觉失衡，大图过于突兀。
2. 子项目截图只有单张静态图，无法承载更多真实功能证据。

## 方案

采用统一 gallery 组件：

1. 主项目区右侧改成一个旗舰画廊
2. 四个子项目截图区改成统一的小型画廊
3. 默认显示一张图，通过左右按钮和圆点切换
4. 单张项目兼容无切换状态

## 设计原则

1. 不做自动轮播，避免干扰阅读
2. 不把所有图片都摊开，避免页面变成截图仓库
3. 保留现有文案与页面整体视觉，只调整展示方式
4. 交互形式克制，服务于求职证据展示而不是炫技

## 图片策略

### 主项目 Cosoul.AI

混合使用：

1. 市场展示图
2. 功能展示图

### 四个子项目

每个项目按 1 到 3 张图组织：

1. `Soul Eval`：评测概览、细节工作区
2. `Humanlike Task Agent Prompt`：多模型对比、Prompt 报告
3. `PipeView`：当前只有 1 张，保留单图兼容
4. `Cosoul.AI-Algorithm-Test`：实验总览、报告页
