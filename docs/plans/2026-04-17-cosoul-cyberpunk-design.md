# Cosoul Cyberpunk Portfolio Design

**Goal:** Rebuild the homepage into a cyberpunk product-release style portfolio that presents the user as an AI engineer with product sense and full-stack delivery capability, centered on the Cosoul project family.

**Audience:** Recruiters, engineering managers, AI product teams, and startup founders hiring for AI engineer / AI application / agent system roles.

**Positioning Statement:** AI工程师，兼具产品与全栈落地能力。

---

## Design Intent

The current Apple-like version is too restrained for the user's taste and underplays the AI / Agent identity. The redesign should feel more futuristic and sharper without becoming game-like or noisy.

The page should read like a launch page for an AI systems builder:

- dark interface
- neon signal colors
- layered panels
- terminal / system-board visual hints
- stronger hierarchy around role identity and system architecture

The goal is not “cool for its own sake”. The visual language must support a hiring narrative:

1. the user can lead or co-drive AI products
2. the user can execute across PM, engineering, and technical experimentation
3. the Cosoul project family forms one coherent story

---

## Content Strategy

### Core Narrative

The homepage should stop presenting the user as only a project maker. It should present the user as:

- AI engineer
- product-minded builder
- full-stack executor
- experimentation-oriented contributor

### Project Priority

The page should heavily prioritize the Cosoul family:

1. Cosoul.AI
2. Soul Eval
3. Humanlike Task Agent Prompt
4. PipeView
5. Cosoul.AI-Algorithm-Test

Other older assets remain in the repo but should not dominate the homepage.

### Contribution Framing

The copy must reflect the user's real contribution pattern:

- Cosoul.AI: team core product with meaningful participation
- supporting Cosoul tools: mostly independent delivery

This matters because the hiring signal is stronger when collaboration and independent ownership both appear clearly.

---

## Page Structure

### 1. Hero

- strong role headline
- supporting line explaining PM + engineering + experimentation blend
- compact role chips such as:
  - AI Engineering
  - Agent Systems
  - Product Execution
  - Full-Stack Delivery
- visual system board using the newly added screenshots

### 2. Mission / Role Frame

- explain how the user works inside AI projects
- clarify that the user is not only a model caller, but someone who can push product, frontend, backend, eval, and algorithm experiments together

### 3. Cosoul Main Story

- deep section for Cosoul.AI
- explain system concept, product structure, cross-platform architecture, and where the user fits
- screenshots should make the section feel like a flagship release panel

### 4. Supporting Systems Grid

- four high-signal cards
- each framed as a system around the main product:
  - evaluation
  - prompt lab
  - monitoring
  - algorithm experimentation

### 5. Capability Matrix

- convert “skills” into capability systems
- suggested axes:
  - AI Engineering
  - Product Thinking
  - Full-Stack Delivery
  - Evaluation and Experimentation

### 6. Career-Oriented Closing

- make the hiring target explicit
- highlight fit for:
  - AI engineer
  - AI application R&D
  - agent system development
  - product-facing AI engineering roles

---

## Visual System

### Palette

- base: deep graphite / near-black
- surface: dark indigo / charcoal panels
- accents: electric cyan, signal magenta, soft violet
- text: cool white / steel gray

### Atmosphere

- faint grid overlays
- scanning lines
- glow borders used sparingly
- luminous section dividers
- “system active” visual language rather than hacker cliché

### Typography

- bold, condensed-feeling headline rhythm
- clean sans body text for readability
- no playful cyberpunk fonts that reduce professionalism

### Motion

- reveal on scroll
- pulse/glow on chips
- panel hover depth
- subtle energy sweep effects on major cards

---

## Risks

- too much cyberpunk styling would weaken recruiter trust
- too much PM wording would dilute the AI engineer identity
- too much engineering wording would hide the user's cross-functional value

The balance should be:

AI engineer first, product/full-stack/execution second.
