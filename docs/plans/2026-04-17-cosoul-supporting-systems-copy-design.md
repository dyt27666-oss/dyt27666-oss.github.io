# Cosoul Supporting Systems Copy Design

日期：2026-04-17

## 目标

将首页中四个 `Cosoul` 支撑项目的文案从“功能介绍型”改为“简历战绩型”，统一为更适合招聘方阅读的表达结构。

## 问题

当前四个子项目虽然已经有真实截图，但文字仍偏向“项目是什么”和“做了哪些功能”。这会带来两个问题：

1. 招聘方能看到系统，但不一定能快速理解候选人在其中的角色
2. 主项目已经强化成“个人推进能力”表达后，子项目如果还停留在功能说明，会造成叙事断裂

## 方案

四个项目统一改写为以下结构：

1. 为什么做：该系统是为主产品解决什么问题
2. 我负责什么：候选人在项目中的核心动作
3. 它说明什么：该项目证明的能力标签
4. 保留精简技术标签，避免堆砌

## 改写原则

1. 不写成普通 CRUD 或 demo 介绍
2. 不夸大为“全栈全包完成一切”
3. 用招聘语言解释项目价值，而不是只解释技术实现
4. 强调复合型能力：
   - AI 工程
   - 产品推进
   - 全栈落地
   - 评测与实验

## 预期结果

改完后，四个子项目应该能分别支撑如下能力画像：

1. `Soul Eval`：评测与优化闭环
2. `Humanlike Task Agent Prompt`：Prompt 设计与交互体验优化
3. `PipeView`：系统可观测性与流程分析
4. `Cosoul.AI-Algorithm-Test`：匹配算法实验与分析
